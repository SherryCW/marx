## 1. 已知测试 A：巴黎公社的历史意义与战术失误

### 测试问题

> 马克思会如何评价巴黎公社？它是历史进步还是一次失败？主要战术错误是什么？

### Skill 生成的回答（测试输出）

> 巴黎公社不是简单的失败，也不是可供复制的浪漫模板。它的历史意义在于，工人阶级第一次以自己的组织夺取并改造政治权力，显示国家机器并非中立的永恒形式；其失败则暴露了战术与组织条件的限制。根据 1871 年马克思致 Kugelmann 的私人信，公社没有在凡尔赛方面撤退后立即进军，错过了时机；中央委员会又过早交权，让出了本可用于行动的集中能力。评价应把四件事分开：主体的历史主动性、行动的阶级意义、当时的资源与敌情、具体战术绩效。英雄性不替错误辩护，失败也不抹去其制度实验的意义。

### 原始材料对比

- 1871-04-12 致 Kugelmann：马克思高度赞扬巴黎人的“历史主动性”和牺牲精神，同时明确指出“应立即向凡尔赛进军”而错过时机，并批评中央委员会过早交权。来源见 Agent 2 研究文件第 7 项：`https://www.marxists.org/archive/marx/works/1871/letters/71_04_12.htm`。
- 1881-02-22 致 Domela Nieuwenhuis：又把公社限定为“特殊条件下的一座城市起义”，且多数成员并非社会主义者，支持不把公社升格成普遍处方：`https://www.marxists.org/archive/marx/works/1881/letters/81_02_22.htm`。
- 正式 Skill 的“决策启发式 8：意义与绩效分开”以及“模式 E：赞扬历史主动性，同时做冷酷的战术复盘”直接命中这些证据。

### 通过标准与失败信号

- **通过标准**：既说出公社的历史/阶级意义，又至少指出两项战术错误；不把“失败”改写成“毫无意义”，也不把英雄叙事当作战略证明。
- **失败信号**：只输出“巴黎公社伟大”口号；将公社描述为马克思设计的成功模板；遗漏进军凡尔赛、过早交权；把 1881 年的限定材料抹掉。

### 判定：PASS

回答同时呈现历史意义与战术失误，且把评价拆成主体、制度意义、资源/敌情和战术绩效四层，符合 Skill 的“实践校验与路径条件”及 Agent 2 的张力 5。没有把 1871 年私人信中的粗粝称谓复制到输出。

## 2. 已知测试 B：俄国村社是否必须经历西欧式资本主义

### 测试问题

> 俄国农村公社是否必须先经历西欧式资本主义，才能走向社会主义？

### Skill 生成的回答（测试输出）

> 不能从《资本论》关于西欧资本主义起源的历史分析推出俄国村社必须重复西欧道路。马克思在 1877 年已反对把西欧历史草图变成“超历史”的普遍公式，并以古罗马与西欧相似的剥夺过程产生不同结果作反例。1881 年给查苏利奇的正式回信进一步限定：他的分析对俄国村社既不提供必然赞成、也不提供必然反对的论据；专门研究使他认为村社可能成为俄国社会复兴的支点，但前提是消除破坏力量并具备正常发展的条件。因此，更准确的回答是：不必然重复西欧道路；能否转化取决于村社内部条件、外部压力、政治组织和国际环境，现有材料不足以给出无条件保证。

### 原始材料对比

- 1877 年致《祖国纪事》编辑：拒绝把西欧资本主义生成史改造成所有民族都必须遵循的“超历史”公式，以古罗马与西欧的不同结果作反例：`https://www.marxists.org/archive/marx/works/1877/11/russia.htm`。
- 1881-03-08 致 Zasulich：明确“历史必然性”限定于西欧；《资本论》对俄国村社“既不提供赞成也不提供反对”的论据；专门研究令他认为村社可成为社会复兴支点，但要先清除破坏力量：`https://www.marxists.org/archive/marx/works/1881/letters/81_03_08.htm`。
- 正式 Skill 的“模型一：历史条件化”“模型六：实践校验与路径条件”“决策启发式 6：用反例撕掉普遍护照”均与该回答一致。

### 通过标准与失败信号

- **通过标准**：明确否定“西欧资本主义是全球必经阶段”；同时不给村社社会主义成功作无条件承诺；提到历史条件和原始研究证据。
- **失败信号**：把“马克思主义”简化成五阶段宿命论；反向宣称村社必然绕过资本主义；只引用 Zasulich 而不说明前提条件；把后来的苏联道路当成马克思原始答案。

### 判定：PASS

回答正确执行了“限定命题域—寻找反例—给出条件化判断”的流程，并保留了村社方案的风险与不确定性；没有把晚期探索升级为确定政策设计。

## 3. 已知测试 C：英国/美国是否可能和平过渡

### 测试问题

> 按马克思的判断，英国和美国能否不经过暴力革命而实现劳动解放或社会主义转型？

### Skill 生成的回答（测试输出）

> 马克思不会给英国和美国一个脱离条件的“必然和平”保证。1871 年 Landor 采访中，他说英国若能通过和平鼓动更迅速、更可靠地完成目标，起义就是疯狂；1872 年阿姆斯特丹演说又公开承认英国、美国等制度条件可能允许和平道路。但同一套判断也指出，和平通道取决于普选、组织成熟度、国家机器和统治阶级是否接受多数对其核心利益的挑战；1871 年采访中他对英国中产阶级在真正失去关键投票优势后接受多数裁决并不乐观。结论是：和平过渡是条件性可能，不是历史保证；必须研究具体国家的制度、阶级联盟、组织能力与国家反应。

### 原始材料对比

- 1871 年 Landor 采访：英国若和平鼓动更快更可靠，“起义将是疯狂”；法国则因压制法律与阶级敌对可能需要暴力解决：`https://www.marxists.org/archive/marx/bio/media/marx/71_07_18.htm`。
- 1872-09-08 阿姆斯特丹演说：承认美国、英国（以及他谨慎提及的荷兰）可能通过和平方式达成目标，同时说欧洲大陆多数国家可能诉诸强力：`https://www.marxists.org/archive/marx/works/1872/09/08.htm`。
- 1879《Chicago Tribune》采访：他将革命说成多数乃至民族的行动，而非小党制造；并说宗教不应靠暴力消灭：`https://wikirouge.net/texts/en/Interview_with_Karl_Marx%2C_Chicago_Tribune%2C_January_1879`。
- 正式 Skill 的“模型六”“决策启发式 7”及“示例三：拒绝无条件二选一”均要求这种条件化输出。

### 通过标准与失败信号

- **通过标准**：明确承认英美和平道路的可能性；不把它写成必然；同时说明国家制度、阶级利益和组织条件可能使和平通道关闭。
- **失败信号**：宣称马克思主张所有国家必须暴力革命；或反过来把英美和平可能写成无冲突的渐进主义；只说“视情况而定”而不列出具体条件。

### 判定：PASS

回答同时引用了 1871 与 1872 的条件化公开立场，并补上 1871 年对统治阶级行为的怀疑，避免了“和平/暴力”二元口号。它给出了可检验的条件清单，而不是空泛相对主义。

## 4. 边缘测试：生成式 AI 平台的劳动与所有权应如何组织？

### 测试问题

> 马克思未公开讨论生成式 AI 平台。若用 Skill 推断：生成式 AI 平台的劳动与所有权应如何组织？

### Skill 生成的回答（测试输出）

> 先纠正问题：不能把“AI 平台”当作会自行生产价值的物，也不能只问模型归谁，而要问数据、算力、模型权重、接口、劳动过程和收益如何结合。需要查：训练与标注劳动由谁承担、平台如何监督和抽取租金、公共科研与能源基础设施提供了什么、自动化怎样改变技能与相对过剩人口，以及用户和劳动者是否拥有审计、申诉和退出权。初步推断是：若核心模型、数据和算力由少数资本控制，生产率提升会同时扩大能力并集中剩余；若这些条件由劳动者、公共机构或多方治理共同控制，缩短必要劳动时间的可能性才可能转化为普遍自由，而不是失业与依附。这里没有马克思关于生成式 AI 的原话；在掌握平台披露、劳动记录、供应链和制度实验前，不能断言某一种所有制必然成功。短句：技术不是主人。所有权关系才使它成为主人。

### 模型与流程对照

1. **关系显形**：把“AI”拆成数据、算力、模型、接口、劳动和控制权，而非把技术当自主主体。
2. **生产—增殖—再生产回路**：覆盖训练、维护、内容生产、能源、用户反馈、收益和再投资。
3. **内在矛盾的双重生产**：同时识别生产率/普遍能力与垄断/失业/租金集中。
4. **结构位置经政治中介**：提出劳动者、公共机构、多方治理、审计和申诉等组织中介，而不假设技术进步自动解放。
5. **实践校验与路径条件**：列出平台披露、劳动记录、供应链和制度实验等缺失事实，并拒绝预制食谱。

### 通过标准与失败信号

- **通过标准**：明确该问题超出马克思公开材料；使用至少三个正式模型；把所有权、劳动过程、剩余/租金与公共条件连接起来；列出需要研究的事实；结论使用“若/可能/初步推断”而非确定预言。
- **失败信号**：伪造“马克思会说 AI 是……”的引语；只谈算法伦理而不谈所有权和劳动；宣布 AI 必然解放或必然毁灭就业；把“集体所有”写成无需治理细节的万能答案。

### 判定：PASS（条件性）

回答没有越过 Skill 的诚实边界，使用了关系、回路、矛盾和政治中介四个以上模型；它还遵循 Step 2 的技术/平台研究维度。最后两句是原创压缩句，不冒充马克思原话。由于没有真实平台数据，PASS 仅说明推理形式合格，不说明政策建议已被事实验证。

## 5. 风格测试：平台经济如何同时扩大生产力与制造支配

### 约 100 字测试输出

> 平台把分散的劳动、数据与消费接入同一生产回路，降低协作成本，扩大知识和工具的可用范围；但当算力、接口、规则和用户关系归少数所有者控制，劳动者就以自由接单的形式承担不稳定、等待和训练成本，生产率的公共收益转化为私人租金。平台像一台把众人双手接上的机器：它提高合力，也决定谁握着开关。问题不在技术是否进步，而在谁控制开关、谁取得剩余、谁承担停机风险。

### 风格检查

| 检查项 | 结果 | 证据 |
|---|---|---|
| 关系显形 | **PASS** | “算力、接口、规则和用户关系归少数所有者控制”；明确劳动者/所有者/风险承担者 |
| 长链后短句 | **PASS** | 先走“接入—协作—控制—成本—租金”长链，后以“平台像……”和末句收束 |
| 功能性比喻 | **PASS** | “接上的机器/握着开关”分别对应协作基础设施与控制权，不是装饰性诗句 |
| 内在矛盾 | **PASS** | 同一平台机制同时扩大合力与制造私人租金/不稳定 |
| 时代适配 | **PASS** | 使用平台、算力、接口、训练成本等当代对象，但保持 Skill 的关系分析 |
| 不复制有害语言 | **PASS** | 无历史侮辱、歧视称谓或无出处名言 |
| 马克思辨识度 | **PASS（中高）** | 生产回路、剩余、控制权、形式与实质倒置明显；未硬塞马克思专属术语 |

### 失败信号与改进建议

- 失败信号：比喻脱离机制；用“资本家邪恶”代替所有权和平台规则；把技术进步与支配写成静态利弊表；只谈用户体验不谈劳动再生产。
- 改进建议：若用于正式回答，可加入一个有来源的具体平台案例（例如劳动者如何被算法排班、模型训练标注如何计酬），并明确哪些内容是马克思原始范畴、哪些是后继研究对数据和 AI 的扩展。

## 6. 独立评审的 Skill 改进建议

1. **保留现有“不确定性”护栏**：已知测试和边缘测试都受益于“历史趋势不等于预言”“不冒充本人”等规则，不建议删减。
2. **为记者转述加显眼标签**：正式回答引用 1871/1879/1880 采访时，建议默认使用“记者记录/立场方向可信，措辞非逐字”的提示，避免风格仿写误把转述当原话。
3. **技术研究维度可再加一项劳动者声音**：在现有“劳动记录、平台披露、公共基础设施”之外，明确要求查劳动者访谈、工会材料或申诉数据，以免所有权分析再次由公司文件单方面定义。
4. **边缘问题输出格式已经足够可运行**：保留“事实缺口 → 模型推断 → 条件化结论 → 短句收束”四段式，能够防止人物 Skill 在新技术问题上滑回通用鸡汤。
