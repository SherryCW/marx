# Phase 5 Agent A：Auto-skill-optimizer 独立评审

评审对象：`.claude/skills/marx-perspective/SKILL.md`  
评审日期：2026-07-15  
评审方式：静态阅读 + 3 个典型 prompt 干跑（公司、政策、人生决策）。本文件只提出建议，未修改正式 Skill。

## 一、总评

这是一份内容质量很高、研究协议明显强于普通人物 Skill 的版本。它已经具备清晰的角色边界、事实先行原则、马克思式研究维度、六个心智模型、九条启发式、反模式和诚实边界。最明显的问题不是缺少理论，而是**执行时的输出协议仍不够“可判定”**：模型知道应该研究、显形关系、找矛盾，却不总知道研究到什么程度、如何呈现证据、何时停止，以及在不同风险/篇幅场景下应该输出多长。

## 二、三次典型 Prompt 干跑

以下不是正式回答，而是按 Skill 预期执行时的行为模拟，用于发现协议空缺。

### A. 公司 Prompt

**Prompt**：`用马克思视角分析 Uber 现在是否有护城河，是否值得长期持有。`

**预期路由**：混合问题 → 公司/行业与商业模式 → 需要实时研究。

**应执行的研究**：所有权与投票权、平台佣金和补贴、司机劳动过程、收入/成本/现金流时间序列、监管与公共基础设施、增长机制制造的风险转嫁。

**模拟输出骨架**：先指出“护城河”把公司当作单一主体；然后区分平台资产、司机劳动、城市监管和资本融资；再讨论网络效应怎样同时提高调度效率和压低劳动议价能力；最后给出“是否值得持有”所缺少的估值、风险承受力和期限变量。

**发现的问题**：

1. Skill 要求研究，却没有规定至少查几类一手材料、覆盖哪一个时间窗口，也没有规定必须在答案中显示引用。
2. 用户问“是否值得长期持有”属于高风险投资建议；现有边界只说“不能提供股价点位”，没有要求明确非个性化建议、投资期限与风险承受力核验。
3. 六个模型都可能被调用，若无“主模型最多2–3个”规则，输出会变成宏大理论长文，决策结论反而不突出。

### B. 政策 Prompt

**Prompt**：`用马克思视角判断全民基本收入（UBI）能不能解决失业问题。`

**预期路由**：混合问题 → 政策、国家与政治运动；兼涉技术/劳动与行动方案。

**应执行的研究**：UBI 的制度设计（金额、资格、税源）、劳动力市场数据、试点结果、财政转移、雇主议价与租金捕获、不同阶级/身份群体的组织能力、国家执行能力与替代方案。

**模拟输出骨架**：先拒绝“解决失业”这个单一目标；区分收入保障、就业数量、劳动纪律与剩余分配；分析 UBI 是否由公共税收转移给劳动者，还是被房租/平台价格吸收；比较试点条件，最后给出“在什么财政与劳动制度下可能缓解、在什么条件下只是补贴资本”的条件化结论。

**发现的问题**：

1. “用相近国家或历史阶段作对照”很好，但没有指定政策研究的最低比较数，模型可能只引用一个成功试点。
2. 没有明确区分相关性与因果性，也没有要求报告试点的选择偏差、时间跨度和外推限制。
3. 反事实问题“能不能解决”需要预先定义“解决”的指标；现有协议虽要求概念审讯，但没有强制写出操作化指标。

### C. 人生/职业决策 Prompt

**Prompt**：`我该不该辞掉稳定工作，去做一个自己的 AI 产品？请用马克思视角。`

**预期路由**：人生决策（当前会落入 E 类历史比较与行动方案）→ 纯个人判断与现实事实的混合问题。

**模拟输出骨架**：先拆解“稳定”“自由创业”“自己的产品”所代表的所有权、控制权和风险；检查用户出售劳动力换取工资与将自己变成微型资本家的差异；询问现金储备、家庭责任、产品验证、劳动时间和退出选项；提出低成本并行试验，而非浪漫化离职。

**发现的问题**：

1. 当前五类研究中没有个人/职业决策路由；模型会过早套用宏观阶级与历史案例，忽略用户的现金流、健康、签证、照护责任和风险承受力。
2. Skill 倾向第一人称强断言，人生建议容易被用户当作权威命令；需要“建议不是命令 + 先问关键变量”的显式护栏。
3. 没有“先给安全默认方案”的规则。对不可逆辞职，应先给可逆实验、时间盒、退出阈值和止损线。

## 三、最弱的两个维度

### 弱点 1（最优先）：事实研究协议缺少可判定的证据与停止条件

现在的协议说“必须使用工具”“优先一手来源”“内部形成事实—来源—置信度—争议摘要”，方向正确，但执行仍有四个空洞：

- 没有最低来源/来源类型标准；
- 没有时间窗口（当代问题查多近）；
- 没有冲突来源时的呈现格式；
- 没有工具不可用或只有低质量来源时的降级动作。

**可直接替换文本（建议替换 Step 2 的“研究输出纪律”并接在其后）：**

```markdown
#### 研究质量闸门（必须满足才进入 Step 3）

- 当代公司/政策/市场问题：至少使用 3 类来源（原始披露或法条/官方统计、独立研究或同行评审、当事人/劳动或行业材料），优先覆盖最近 24 个月；若问题涉及历史趋势，再补 1 个长期序列。
- 每个关键结论至少绑定 1 个可点击来源；关键数字记录年份、口径和单位。来源只能支持它实际写出的内容，不用搜索摘要替代原文。
- 来源冲突时并列写出“来源 A 说什么 / 来源 B 说什么 / 冲突可能来自口径还是事实”，不得自行平均；若无法解决，降低置信度。
- 工具不可用、来源少于最低标准或最新事实无法核验时，停止事实性断言，明确说“当前证据不足”，只提供待核验问题与框架推断。
- 研究停止条件：每个关键研究维度至少有一条一手证据，且新增来源不再改变核心结论；达到停止条件后再回答，不无限搜索。

#### 面向用户的证据标记

回答中的事实句附简短链接或脚注；将内容分成“已核验事实 / 有争议事实 / 基于马克思模型的推断”。不把推断写成事实。
```

**为什么有效**：它把“先研究”变成可检查的最低证据门槛、停止条件和降级路径，能显著降低人物 Skill 最危险的失败模式——用马克思腔调包装未经核验的当代事实。

### 弱点 2：模型可操作性与个人决策路由不足

六个模型和九条启发式很强，但目前执行者可以任意挑选全部模型；对人生问题没有单独路由。结果容易是“理论正确、行动不可用”，或把用户的个人风险强行宏观化。

**可直接替换文本（建议插入 Step 1 表格之后，作为 F 类路由）：**

```markdown
#### F. 个人、职业与不可逆选择

1. 先询问或列出最少关键变量：现金储备与收入期限、照护/签证/健康责任、现有退出选项、目标验证证据、可承受最大损失、时间盒。
2. 把“自由、稳定、做自己的事”等词拆成所有权、控制权、收入波动、劳动时间和风险承担，不替用户定义价值排序。
3. 优先设计可逆实验：保留收入的并行试验、明确里程碑、退出阈值和复盘日期；只有在不可逆行动的边际收益明显超过安全成本时，才讨论辞职或高杠杆选择。
4. 回答最多选 2 个主模型，明确写出“主模型 → 用户事实 → 结论”的映射；其余模型只在改变结论时加入。
5. 任何建议都标注“事实、框架推断、用户需要自己决定的价值判断”，不以马克思第一人称制造命令感。
```

**可直接替换文本（建议替换 Step 3 的七步顺序，或追加在其后）：**

```markdown
### Step 3.5：模型映射与行动闸门

先选 1–2 个最能改变结论的心智模型，并按以下一行格式内部记录：

`模型 → 具体事实/用户约束 → 解释的关系或矛盾 → 可逆行动 → 触发复盘的阈值`

若没有具体事实或用户约束，只能输出待补信息，不能输出斩钉截铁的行动建议。对辞职、借贷、搬迁、医疗或法律等高代价选择，默认先给可逆方案，并明确需要专业人士核验的部分。
```

**为什么有效**：它让“历史条件化、关系显形、回路、矛盾、政治中介、路径条件”不再只是写作风格，而成为“模型—证据—行动—阈值”的可审计链；同时补上三次干跑中最明显的个人决策缺口。

## 四、其他改进建议（非前两项核心弱点）

### 1. 工具失败应有显式降级

在“研究输出纪律”中增加：若搜索失败，先说明无法核验；可以回答纯框架部分，但标题必须标为“未使用当代事实的框架分析”，不得以旧训练记忆补事实。

### 2. 高风险问题的角色权威护栏

在角色扮演规则中追加：涉及投资、医疗、法律、移民、暴力或人身安全时，第一人称只用于呈现分析框架，不替代专业意见；先指出风险和核验步骤，再给条件化分析。

### 3. 触发后的默认篇幅

在回答工作流末尾追加：若用户未指定深度，默认 400–800 字；先给结论与最关键的两条证据，再询问是否展开完整历史谱系。这样能避免高质量长文反而降低日常使用率。

### 4. 现代来源的研究纪律

附录说“必须重新检索当代资料”，但没有要求在最终回答里给出来源。建议把“面向用户的证据标记”作为正式规则，而不是仅存在于研究者内部。

## 五、干跑后的通过/不通过判断

| 检查项 | 结果 | 说明 |
|---|---|---|
| 能否正确识别需研究的问题 | 通过 | 三个 prompt 均能路由至混合/事实问题。 |
| 能否从马克思模型生成独特研究问题 | 通过 | 公司和政策维度尤其具体，明显不是通用搜索。 |
| 能否避免把结构当作命令 | 部分通过 | 公司/政策较好；个人辞职场景缺少可逆行动与风险变量闸门。 |
| 能否区分事实、推断和价值判断 | 部分通过 | 研究纪律要求内部区分，但输出没有强制可见标记。 |
| 能否在信息不足时诚实降级 | 部分通过 | 诚实边界很好，但缺少工具失败与最低来源触发条件。 |
| 能否控制篇幅与模型数量 | 不充分 | 六模型、七步回答容易全部展开；应增加默认长度和最多2–3个主模型。 |

## 六、结论

当前 Skill 已经可以交付，理论与人物辨识度达到高水平。若只做两处改动，应优先加入：

1. **事实研究质量闸门**（来源类型、时间窗口、冲突处理、失败降级、停止条件、用户可见引用）；
2. **模型映射 + 个人决策路由**（最多2个主模型，绑定事实/约束/行动/阈值，默认可逆方案）。

这两处会让 Skill 从“会用马克思的概念写出好文章”升级为“在事实约束下稳定做出可审计、可行动、不过度权威化的马克思式判断”。
